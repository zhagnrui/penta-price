# Gemini版本 UI改造方案 v1.0

## 📐 设计原则

**核心：成本优先 → 技术驱动 → 优化闭环**

```
用户进门
    ↓
看到成本数字（大、清晰）
    ↓
理解成本从哪来（成本分解图）
    ↓
知道这个成本选择的技术含义（技术影响卡片）
    ↓
看到有什么优化机会（优化建议）
    ↓
快速决策（成本-技术矩阵）
```

## 🧩 核心组件

### 1. CostSummary（成本总览）
**位置**：仪表板顶部  
**大小**：全宽  
**内容**：
- 大字体单位成本 ¥2500/吨
- 趋势指标（↑2.5% 对比上月）
- 总成本估算

**颜色**：白底 + 渐变背景（slate-50 to slate-100）

### 2. CostBreakdownChart（成本分解）
**位置**：成本总览下方  
**类型**：条形图（不引入新chart库，纯HTML进度条）  
**交互**：
- 悬停显示具体金额
- 点击可展开该原料的详细信息

**示例数据**：
```
APP 40% ¥1000/吨  ████████████████░░░░
PER 32% ¥800/吨   ██████████░░░░░░░░░░
MEL 28% ¥700/吨   █████████░░░░░░░░░░░
```

### 3. TechImplicationCard（技术影响）
**位置**：成本分解下方  
**显示个数**：3-4张  
**优先级**：info(蓝) > warning(琥珀) > critical(红)  
**内容结构**：
```
┌─ 标题 (粗体)
├─ 成本驱动：[哪个原料导致] ← 关键！必须关联成本
├─ 技术影响：[具体后果]
├─ 💡 解决方案：[怎么办]
└─ 成本影响：[节省多少/增加多少]
```

**示例**：
```
⚠️ APP成本占比40%（最高）
成本驱动：APP ¥1000 (占全部40%)
技术影响：聚磷酸铵是催化剂，含量高决定了阻燃效能
💡 解决方案：可试用P₂O₅ 70%的低端等级，需配合工艺调整
成本影响：可节省¥200-300/吨（但需验证性能）
```

### 4. OptimizationCard（优化建议）
**位置**：技术影响卡片下方  
**显示个数**：2-3张  
**背景**：绿色渐变  
**内容**：
```
┌─ 标题：优化方向
├─ 节省金额：¥250（-10%）
├─ 成本对比：¥1000 → ¥750
├─ 技术权衡：阻燃效能下降8%
├─ 实施方式：选国产APP + 增加MEL用量
└─ [按钮] 采用此方案
```

**特点**：每个优化都要说明技术代价（不是无代价节省）

### 5. CostTechMatrix（决策矩阵）
**位置**：页面中部  
**行数**：3行（经济版/标准版/高端版）  
**每行显示**：
- 价格区间（大字体）
- 描述标签
- 原料选择（具体型号）
- 技术风险列表
- 工艺方案
- 适用场景

**交互**：
- 点击某行可展开详细的工艺指导
- 可选择某行作为"当前方案"（高亮显示）

### 6. SOPChecklist（工艺检查清单）
**位置**：页面下部  
**分组**：原料准备 / 混合工艺 / 固化过程（3列）  
**每项**：可勾选的复选框  
**特点**：从成本选择推导出来（不同成本档位有不同的SOP）

## 🎯 交互流程

### 场景1：用户想降低成本

```
1. 看到当前成本¥2500
2. 查看成本分解 → 发现APP占40%（最贵）
3. 点击APP卡片 → 展示优化建议"降级APP纯度"
4. 查看技术权衡 → 了解性能会降8%
5. 查看决策矩阵 → 发现"经济版¥1800"的方案
6. 点击"采用此方案" → 自动更新所有参数和SOP
```

### 场景2：用户想了解某个选择的风险

```
1. 看到某个技术影响卡片（如"PER纯度"）
2. 点击卡片 → 展开详细说明
3. 看到"这个选择会影响性能X%"
4. 看到"建议做Y工艺调整"
5. SOP清单自动标记相关项目为"必做"
```

### 场景3：用户想对比方案

```
1. 查看成本-技术矩阵
2. 看到经济/标准/高端三种方案并排
3. 每种方案显示原料、风险、工艺
4. 可同时查看三种方案的SOP差异
```


## 📊 UI信息架构对比

### 旧Gemini版本
```
输入 → 计算 → 显示成本 + 成本分解图 + SOP建议
            （单向流）
```

### 新改造版本
```
成本总数（显眼）
    ↓
成本分解（你的钱花在哪）
    ↓
技术影响（这个选择的后果）
    ↓
优化建议（怎么改更好）
    ↓
决策矩阵（快速选择）
    ↓
SOP清单（怎么生产）
    （闭环流）
```

**关键改变**：从"告诉你成本是多少"升级到"告诉你为什么这个成本，有什么办法改"

## ✨ 关键创新点

1. **成本-技术双链路**  
   不是"列举风险"，而是"从成本选择推导风险"

2. **可视化成本驱动**  
   用颜色、大小、位置强调"这个选择最影响成本"

3. **正向激励优化**  
   不是"你有风险"，而是"你可以省这么多钱，代价是..."

4. **快速决策矩阵**  
   用户不需要复杂计算，看矩阵直接选档位

5. **工艺追踪**  
   SOP是从成本选择推导的，不是固定的

## 💬 设计意图

这个UI重构的核心是：**让技术员、研发人员和采购经理都能从成本决策中看到自己关心的信息**。

- **采购经理**：看成本数字和节省方案
- **技术员**：看技术风险和SOP要求  
- **研发**：看性能权衡和优化空间

一个页面，满足三个角色的信息需求。
